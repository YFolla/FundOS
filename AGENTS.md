# Agent Instructions

Start with [TEAMOS.md](/Users/yannickfolla/CascadeProjects/TeamOS/TEAMOS.md), then navigate through the relevant top-level index before opening content files.

## Repo Rules

- Keep root files small and navigational.
- Treat YAML indexes as the primary retrieval layer.
- Do not create duplicate dossiers for the same company, theme, or fund.
- When adding a company, update `companies/companies-index.yaml`.
- When adding a market theme, update `markets/themes-index.yaml`.
- When adding a fund artifact, update `funds/funds-index.yaml`.
- Keep canonical knowledge in Markdown and YAML rather than agent-specific folders.

## Preferred Navigation

1. Root map: `TEAMOS.md`
2. Top-level folder index
3. Relevant YAML object index
4. Linked object files only

## Working Bias

- Prefer the smallest correct diff.
- Preserve source references to Box, CRM, and meeting systems.
- Keep fictional sample data clearly separated from real firm data during early setup.
